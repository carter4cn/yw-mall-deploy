## 踩到的坑

### db-init 容器名冲突

`podman-compose up -d <service>` 会顺带尝试创建 db-init 容器，但它是 `restart: "no"` 的一次性 job，**已 Exited(0)** 的容器名占用导致：

```
Error: container name "yw-mall-deploy_db-init_1" is already in use by ...
```

依赖关系又禁止直接 `podman rm db-init`（其它服务用 `depends_on: db-init`）。

**现行 workaround**：每次要重启 P2 服务都用 `podman rm -f <svc>_1 && podman-compose up -d <svc>` —— 显式跳过 db-init 重建。

**长期方案待办**：把 db-init 改成 `profiles: [bootstrap]`，平时不参与 `up`；首次部署 `--profile bootstrap up db-init` 一次性跑完。
